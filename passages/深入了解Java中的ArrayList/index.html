ArrayList简介

对于Java开发人员来说，ArrayList是最常用的数据结构之一。它是List接口的一个具体实现类。ArrayList是容量可以改变的非线程安全集合。内部实现使用数组进行数据存储，集合扩容时会创建更大的数组空间，在将原有数据复制到新数组中。ArrayList实现了RandomAccess接口从而支持对元素的快速随机访问，但是插入与删除时速度通常较慢（因为删除和插入的过程可能需要移动元素）。ArrayList允许null元素存入，ArrayList的iterator和listIterator方法返回的迭代器是fail-fast的。

fail-fast机制

fail-fast机制是Java集合（Collection）中的一种错误机制，从名字上就可以看出是一种快速进入失败的机制。当多个线程并发对一个集合的内容进行操作的时候，就有可能产生fail-fast事件。请注意，是有可能产生。例如：当某一个线程A通过迭代器Iterator去遍历某集合的过程中，若该集合的内容被其他线程所修改了。那么线程A访问集合的时候，就有可能抛出ConcurrentModificationException异常，产生fail-fast事件。迭代器的“快速失败”行为并不是完备的机制，而是尽最大努力抛出并发修改异常（ConcurrentModificationException）。另外一个需要注意的问题是，单线程的情况下如果违反了规则，同样可能引起fail-fast事件
可以认为这种机制是一种预警指示器~

fail-fast要点总结
  1. fail-fast机制是Java集合（Collection）中的一种错误机制，它不是完备的检查机制，而是尽最大努力去做检查。
  2. fail-fast的触发仅仅在于多线程情况下对Collection进行修改，单线程情况下使用不合理一样会触发
  3. java.util包中的集合类都返回fail-fast迭代器，这意味着他们假设线程在集合内容中进行迭代时候，集合不会更改它的内容，但是如果发生了修改，它会抛出ConcurrentModificationException，这是不可控的。
  4. 并发情况下应该改用java.util.concurrent下的集合
  5. Iterator 在工作的时候是不允许被迭代的对象被改变的。但你可以使用 Iterator 本身的方法 remove() 来删除对象，而不是使用Collection本身提供的remove方法。
