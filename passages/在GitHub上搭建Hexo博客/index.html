在GitHub上搭建Hexo博客

Hexo简介

通过GitHub创建一个仓库github.io结合HEXO就能快速搭建一个能够访问的博客站点。Hexo是一个
快速上手，简单高效，支持多种主题的博客系统（基于Node.js）。通过Hexo创建文章后，再用Markdown语法对文章进行编辑，
后发布到github.io仓库，即可访问该文章。

快速入门

Step One:创建.io仓库

1.New repository

2.在Repository name输入框，必需为YourUserName.github.io格式。YourUserName替换为你的用户名

3.Create repository

4.进入仓库YourUserName.github.io,点击setting,找到GitHub Pages模块

5.点击choose a theme选择一个页面主题

6.访问https://username.github.io就可以访问博客网站了

Step Two:安装Git

下载地址：https://www.git-scm.com/download/
推荐教程：廖雪峰老是的Git教程

Step Three:安装Node.Js和Hexo

Hexo是基于node.Js环境的静态博客系统，因此在安装之前需要先装node.Js
有的人会遇到node.Js版本过低导致Hexo安装失败，所以这里建议尽量选版本高点的。

检验是否安装Node.Js成功

Step Four:创建本地博客站点

1.创建一个文件夹,如blog
2.进入blog目录并执行指令hexo init 初始化博客站点(执行结果是：Start blogging with Hexo!则创建成功)
3.执行hexo server启动博客站点，并在http://localhost:4000访问
4.Crtl+C关闭服务

Step Five:本地Hexo关联Github

1.进入blog目录 执行安装 npm install hexo-deployer-git –save
2.找到blog目录下的_config.yml文件，修改最底部的Deployment配置，如下所示。

注意：yml有严格的格式要求，冒号后需要+上一个空格
3.将本地Hexo部署到Github上，执行如下指令 （首次上传可能需要你输入登陆github信息）

hexo clean
hexo generate (也可以用hexo g)
hexo deploy (也可以用hexo d)

4:验证是否成功关联，打开http://YourUserName.github.io

Step Six:创建和发布文章

1.进入blog目录后，使用hexo new “文章标题”指令，创建MD文件
2.在/source/_posts中可以看到刚刚创建的MD文件，用编辑器打开(也可以用有道云笔记打开，现在支持Md语法了)
3.编辑后保存，执行hexo d -g发布文章,首次使用会要求输入git账号信息

其他技巧

如何给文章引入图片

背景：默认无法自动处理文章插入本地图片，需要通过扩展插件支持。

1:_config.yml里面的post_asset_folde选项设置为true。

2:安装插件hexo-asset-image，安装以后 hexo n “YourPost”指令在生成MD文件的同时会生成一个名字与MD文件相同的文件夹，将本地图片放置在此，然后在文章通过格式 ![xxx](xxx/xxx.png) 引入图片即可。

npm install https://github.com/7ym0n/hexo-asset-image –save

更换电脑了，如何迁移文章

背景：程序员经常都会换生产工具，需要对文章进行备份
解决这个问题很简单，只要把原有的_post文件夹里的内容完整copy到新机子Hexo对应的_post下即可。
