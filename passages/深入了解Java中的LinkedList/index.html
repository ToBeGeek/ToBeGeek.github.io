前言

ArrayList和Vector在底层的实现上都是根据数组实现的，他俩的实现原理非常接近。而此次要总结的LinkedList在底层上是基于（双向）链表实现的。也就是说在内存空间的分配上，LinkedList分配的空间可以是不连续的，仅仅在逻辑意义上连续。大一的时候，用过C语言写链表结构的同学应该都知道。链表是由节点node构成的，每个node都至少包含指向下一个节点的句柄(指针，如果只有当个方向的指针则为单向链表，包含上个node以及下个node的指针则为双向链表)。

数组与链表的比较

在讲解LinkedList之前，先复习复习数组与链表这两个数据结构的特性以及优缺点

数组的特点
  1. 在内存中，数组是一块连续的区域（物理空间是连续&有序）
  2. 数组在使用之前，必须进行初始化，申请内存大小（一般会有预留空间）
  3. 数组在插入和删除元素的时候效率低，插入和删除元素的时间复杂度会受到目标位置的影响。（如果在数组尾部添加元素，则时间复杂度就是O(1),此时是常数级的。但是如果在指定位置i插入或删除元素的话，时间复杂度就是O(n-i),n是数组的长度，n-i的意思表示数组中的第i和第i个元素后的n-i个元素都需要进行挪移位置操作，O(n-i)其实就是线性级的复杂度了）
  4. 支持快速随机访问，且访问效率极高。毕竟数组是连续的，可以根据数组的索引直接找到对应的元素位置
  5. 不利于扩展，当数组长度不满足的时候，只能重新分配新内存空间（也就是重新定义数组，如ArrayList和Vector的实现原理，在每次进行扩容的时候都是调用 Arrays.copyOf(elementData, size, Object[].class)）
链表的特点
  1. 链表是逻辑意义上连续存储的一种数据结构，在物理上没有要求
  2. 链表中的每个节点都至少保存了相邻节点的一个或者两个指针（即每个节点数据都保存了下一个或者上一个节点的内存地址）
  3. 插入和删除元素的效率高且稳定，不会受到元素位置的影响。（就像我们买了个新的手表，手表带过长的时候我们需要把第i块手表带粒拆除，只需要拆掉i块后，将i-1,i+1进行连接，也就是链表中的第i-1节点中的指针指向第i+1节点，对于其他的节点来说不需要做出改动，近似O(1)，也就是常数级的复杂度）
  4. 查找效率低，因为不具有随机访问性，所以每次访问某个位置的时候就必须从链表的端开始顺序访问（还是以上面的第三点为例子，假设本人高度近视且眼镜没戴，那我只能顺着表带的端点，从第一个表带粒开始摸，一边摸一边数，直到第i个位置在进行拆表带粒，那么这个访问的算法复杂度就是O(n),也就是线性级）
  5. 内存利用效率高，扩展方便。链表的内存空间不需要提前申请，数据随意增删（需要的时候再申请）
数组和链表的优缺点以及怎么使用
数据结构 优点 缺点
数组 支持快速随机访问，查找速度快 1.插入和删除元素的效率低
2.容易造成内存浪费，申请的空间超出实际需要
3.数组大小固定，不能动态扩展
链表 1.插入和删除元素效率高
2.内存利用率高，按需分配，不会造成内存浪费
3.扩展灵活		 不支持快速随机访问，需要从端开始遍历查找，查找效率低

不难看出。当我们需要对数据结构中的元素进行频繁的查找时候，我们应该采用基于数组的数据结构。而当我们需要频繁增加或删除元素的时候，我们应该采用链表。
在使用数组这个数据结构之前，我们要尽可能地保证申请地空间大小满足我们的实际需要，从而避免内存浪费。
